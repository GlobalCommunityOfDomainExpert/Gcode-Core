# Icon

## Purpose

Icons provide visual context and improve recognition. They should support content, not replace text.

---

## Icon Library

Use **Lucide Icons** for all interface icons.

Do not mix icon libraries.

---

## Usage

Use icons for:

- Navigation
- Buttons
- Status indicators
- Actions
- Inputs
- Empty states

Avoid using icons as the only indicator of meaning.

---

## Sizes

- Small
- Medium
- Large

Choose the size that matches the surrounding component.

---

## Guidelines

- Keep icons consistent throughout the application.
- Decorative icons should not receive keyboard focus.
- Pair important icons with text labels.
- Use the same icon for the same action across the product.
